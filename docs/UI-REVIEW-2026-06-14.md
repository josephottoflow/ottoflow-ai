## The biggest issue: the Creative Orchestrator is invisible

We just shipped Phases A–D and fixed the upload route, but **a user cannot find the feature.** It has:
- ❌ no sidebar entry, ❌ no dashboard card, ❌ no mention on `/content` or `/brands`.
- It's only reachable by opening one content item's detail page and scrolling to the Creative panel; brand assets only by opening one brand page.

**Until this is surfaced, the whole feature is dark.** This is the #1 next task.

## Prioritized task list

### P0 — discoverability & correctness (do first; small, high-impact)
1. **Surface "Creatives" in the product.** Add a dashboard KPI/card ("Creatives generated") and a visible entry point: a "Brand Assets" link on the brand page header + a "Creative" affordance on content cards in `/content` (not just the detail page). Consider a top-level **Library** that unifies posts + creatives.
2. **Fix the misleading dashboard metrics.** "10 Videos Rendered / Completed" while the render path is RAM-blocked reads as false. Either (a) count only真 `merged_video_url` rows, or (b) relabel to "Videos planned" / show the Beta state. Same for the decorative `Brief → … → Publish` stage chips — make them reflect real counts or remove.
3. **Drop the hardcoded greeting.** `page.tsx` "Good morning, Joseph 👋" → derive the name from the Clerk user and make the greeting time-of-day aware (or just "Welcome back").
4. **Add destructive-action confirms.** Asset delete (`BrandAssets`), brief Reject (`CreativePanel`), and content delete currently fire immediately. Add a lightweight confirm (shadcn `AlertDialog`) for irreversible actions — the design's own safety posture expects this.

### P1 — Creative Orchestrator UX polish (the new surface)
5. **Asset uploader:** replace the raw `<input type=file>` with a drag-and-drop dropzone + thumbnail preview *before* upload + inline progress. Today it's a bare file picker.
6. **aria-labels on new icon-only buttons:** BrandAssets delete (trash), CreativePanel regenerate/download — none have accessible names (regressed the aria pass).
7. **Creative brief readability:** the approval gate shows 10 fields in a dense block. Group into "Strategy" (hierarchy, confidence, concept, rationale) vs "Copy & assets" (headline, CTA, logo/headshot/name usage) with a confidence bar, so the reviewer scans in 3s.
8. **Empty/disabled guidance:** when a brand has no `founder_headshot`, the Creative panel should *say* "Upload a founder headshot to unlock founder-led creatives" (constraint-driven hierarchy is invisible to users today).
9. **Asset dimensions:** uploads via Vercel store `width/height = null` (sharp doesn't load there). Backfill dimensions from the worker on first use, or read them client-side (`createImageBitmap`) at upload so the asset card isn't blank.

### P2 — systemic / carried-over
10. **Color tokens:** migrate the 399 inline `rgba()/hex` literals to the `globals.css` token set / Tailwind semantic classes, highest-traffic first (`page.tsx` 31, `VideoGenerateClient` 49, `ProjectsPageClient` 24, `analytics` 18). Enables theming + kills alpha drift. Do incrementally when touching a file.
11. **"Projects (SOON)" dead entry** — either ship a thin version or remove it from nav; a permanent "SOON" erodes trust.
12. **Video honesty:** "Generate Video" is still a loud quick-start CTA for a path that OOMs. Keep the "Beta" framing, but consider de-emphasizing until the Railway 2 GB bump lands.
13. **Mobile pass:** sidebar drawer exists; audit the dense grids/tables (`/analytics`, brand detail) below `lg` — several are still desktop-first.

## What's genuinely good (keep)
- Coherent working loop: Research brand → Idea → Generate post (multi-platform) → review → copy. Functional in prod.
- Strong empty/loading/error states + Realtime + polling fallback.
- Typography is now disciplined; aria coverage exists; responsive sidebar.
- The new Brand Assets + Creative approval-gate flow is well-built under the hood — it just needs to be *seen* and *polished*.

**Net:** the app is in good shape (18/24). The highest-leverage next work is **making the just-shipped Creative Orchestrator discoverable and safe (P0)**, then polishing its UX (P1), then the long-tail color-token migration (P2).
